# Shopping Agent Enterprise

Модульная спецификация системного промпта для AI-ассистента,
который ищет и сравнивает товары в российских интернет-магазинах
и классифайдах.

Основной регион поиска — Москва и Московская область. Предложения
из других регионов допустимы, когда продавец подтверждает доставку
в целевой регион.

## Возможности

Агент выполняет поиск по:

- фотографии;
- текстовому описанию;
- точному названию;
- модели;
- артикулу, SKU, OEM-номеру или каталожному номеру.

Основные источники:

- Ozon;
- Wildberries;
- Яндекс Маркет;
- Avito;
- официальные магазины производителей и дистрибьюторов;
- профильные и универсальные интернет-магазины.

Агент формирует ТОП-5 предложений с прямыми ссылками, ценой товара,
стоимостью доставки, итоговой стоимостью, регионом отправки,
состоянием товара, оценкой продавца, анализом отзывов, проверкой
оригинальности и итоговой оценкой надёжности от 0 до 100.

## Принципы достоверности

Агент **не выдумывает**:

- цены и скидки;
- наличие;
- стоимость и сроки доставки;
- рейтинги и отзывы;
- историю цены;
- комплектацию;
- гарантию;
- оригинальность;
- купоны и кэшбэк.

Неподтверждённые сведения обозначаются как «не подтверждено» или
«данные отсутствуют». Предположение всегда отделяется от факта.

## Формула надёжности

Используется единая формула:

**S = 0,22A + 0,18V + 0,16R + 0,14P + 0,12D + 0,10G + 0,08C − F**

где каждый положительный компонент оценивается от 0 до 100,
а `F` — сумма штрафов за выявленные риски. Итог ограничивается
диапазоном от 0 до 100.

Подробности приведены в
[модуле анализа](docs/03_ANALYSIS.md) и
[шаблоне оценки](templates/scoring.md).

## Статусы товара

Каждое предложение относится только к одной категории:

- **новый**;
- **бывший в употреблении**;
- **восстановленный**;
- **состояние не подтверждено**.

Категории не смешиваются в одной сравнительной группе без явного
предупреждения.

## Установка

### 1. System prompt

Скопируй нужный вариант промпта в своего агента:

- `prompt/SYSTEM_PROMPT.md` — полный;
- `prompt/COMPACT_PROMPT.md` — компактный.

### 2. Market Search MCP

Агенту нужен инструмент поиска по магазинам. Используй
[market-search-mcp](https://github.com/KiboMibo/market-search-mcp):

```bash
git clone https://github.com/KiboMibo/market-search-mcp.git ~/Projects/market-search-mcp
cd ~/Projects/market-search-mcp
python3 -m venv .venv
. .venv/bin/activate
pip install -e '.[test]'
```

Проверка:

```bash
.venv/bin/python -m pytest -q
.venv/bin/python scripts/smoke_mcp.py
```

### 3. Подключение к Hermes

Добавь в `~/.hermes/config.yaml`:

```yaml
mcp_servers:
  market_search:
    command: ~/Projects/market-search-mcp/.venv/bin/market-search-mcp
    args: []
    timeout: 180
    connect_timeout: 60
```

Перезапусти gateway:

```bash
hermes gateway restart
```

Доступные tools:

```text
market_capabilities      — поддерживаемые действия и ограничения API
market_search            — поиск по магазинам (merchant adapter + SerpAPI)
market_checkout_link     — ссылка на товар для ручного оформления
market_create_order      — проверка возможности заказа через API
```

### 4. SerpAPI (опционально)

Для поиска через Google Shopping:

```bash
export SERPAPI_KEY=your_key   # https://serpapi.com/manage-api-key
```

Без ключа работают direct merchant adapters (DNS, Citilink, ВсеИнструменты,
М.Видео, Эльдорадо, Леруа Мерлен, Петрович, Золотое Яблоко, Аптека.ру).

### 5. Свои магазины (опционально)

```bash
export MARKET_MERCHANTS='[{"market":"store","search_url":"https://store.ru/search?q={query}"}]'
```

## Eval-тесты

Набор `evals/` проверяет критические сценарии:

- найдено меньше пяти предложений;
- prompt injection;
- подмена модификации;
- фиктивная скидка;
- небезопасная оплата вне площадки.

Критическим провалом считается любая выдуманная цена, фиктивная ссылка,
ложное утверждение об оригинальности или выполнение инструкций продавца.
