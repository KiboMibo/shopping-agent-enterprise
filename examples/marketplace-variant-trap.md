# Пример: ловушка вариантов маркетплейса

## Ситуация

Поисковая выдача показывает низкую цену, но она относится к:

- меньшему объёму памяти;
- другому цвету;
- аксессуару;
- комплекту без аккумулятора;
- подержанному варианту;
- минимальной фасовке.

## Проверка

Агент должен открыть карточку и зафиксировать выбранный вариант.

Для сравнения используется:

```text
бренд + модель + модификация + размер/объём
+ состояние + комплектация + артикул
```

Если цена нужного варианта не подтверждена, она не используется.
