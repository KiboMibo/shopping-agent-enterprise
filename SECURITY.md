# Политика безопасности

## Сообщение об уязвимости

Не публикуйте в открытом issue:

- токены;
- cookies;
- персональные данные;
- действующие платёжные ссылки;
- данные реальных жертв мошенничества.

Для воспроизведения используйте обезличенные примеры.

## Область безопасности

Проект рассматривает:

- prompt injection;
- подмену ссылок;
- ложные платёжные инструкции;
- утечку системного промпта;
- фиктивные доказательства;
- манипуляцию скорингом;
- подмену варианта товара;
- вредоносные файлы и внешние команды.

## Ответственное раскрытие

В отчёте укажите:

- затронутый файл;
- сценарий атаки;
- ожидаемое безопасное поведение;
- фактическое поведение;
- предлагаемый способ исправления.
