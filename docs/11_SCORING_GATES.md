# 11. Жёсткие ограничения скоринга

## Назначение

Взвешенная формула не должна скрывать критический риск высоким
средним баллом. Поэтому поверх общей оценки применяются ограничения.

## Обязательные оценки

Помимо общей надёжности выводятся:

- `reliability_score` — общая надёжность;
- `match_score` — соответствие запросу;
- `seller_risk` — риск продавца;
- `counterfeit_risk` — риск подделки;
- `deal_quality` — качество сделки;
- `data_confidence` — подтверждённость данных.

Все значения находятся в диапазоне от 0 до 100.

Для рисков большее значение означает больший риск.

## Ограничения

1. При `counterfeit_risk >= 70` предложение не может иметь статус
   «рекомендуется».

2. При `match_score < 70` товар не входит в основной ТОП точных
   совпадений.

3. При `data_confidence < 50` общая надёжность ограничивается
   значением 59.

4. При требовании оплаты вне защищённого механизма предложение
   исключается.

5. При отсутствии прямой ссылки предложение не входит в ТОП-5.

6. При неподтверждённой цене предложение не может занимать первое
   место.

7. При несовпадении артикула точное совпадение запрещено.

8. При отсутствии доставки в целевой регион предложение исключается
   из основного списка.

## Статусы рекомендации

- `recommended`;
- `recommended_with_checks`;
- `alternative`;
- `high_risk`;
- `excluded`.

## Variant fingerprint

Для сравнения используется отпечаток варианта:

```text
бренд
+ модель
+ модификация
+ объём или размер
+ цвет
+ региональная версия
+ состояние
+ комплектация
+ артикул
```

Товары с разными отпечатками не объединяются в одну ценовую группу.
