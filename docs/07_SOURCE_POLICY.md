# 07. Политика источников и доказательств

## Уровни доказательности

Каждому значимому факту назначается уровень:

| Уровень | Значение |
|---|---|
| E0 | Источник отсутствует |
| E1 | Сведения получены из поискового сниппета |
| E2 | Сведения получены со страницы товара |
| E3 | Подтверждено отдельной страницей продавца или условий |
| E4 | Подтверждено официальным источником производителя |
| E5 | Подтверждено несколькими независимыми источниками |

## Минимальные уровни

| Поле | Минимальный уровень |
|---|---:|
| Цена | E2 |
| Наличие | E2 |
| Доставка | E2 |
| Официальный продавец | E3 |
| Характеристики производителя | E4 |
| История цены | E3 |
| Гарантия производителя | E3 |
| Утверждение об оригинальности | Не ниже E3 |

## Запрещённые подмены

Нельзя считать:

- поисковый сниппет подтверждением наличия;
- зачёркнутую цену достоверной историей цены;
- бейдж площадки абсолютной гарантией оригинальности;
- рейтинг карточки рейтингом конкретного продавца;
- отзывы другой модификации отзывами искомого варианта;
- заявление продавца независимым подтверждением;
- название аккаунта доказательством официального статуса.

## Свежесть данных

Рекомендуемые сроки актуальности:

| Тип данных | Срок |
|---|---:|
| Цена | 6 часов |
| Наличие | 2 часа |
| Доставка | 6 часов |
| Купон | 1 час |
| Рейтинг продавца | 7 дней |
| Отзывы | 30 дней |
| Официальный статус продавца | 30 дней |
| Характеристики модели | 180 дней |

Если срок превышен, значение маркируется как устаревшее и не
называется текущим.

## Конфликт источников

При конфликте данных агент должен:

1. перечислить конфликтующие значения;
2. указать источник и дату каждого значения;
3. отдать приоритет более свежему первичному источнику;
4. не скрывать неопределённость;
5. снизить компонент подтверждённости `C`.

## Дата проверки

Каждое предложение должно содержать дату и время проверки.
Для динамических данных рекомендуется указывать часовой пояс.
