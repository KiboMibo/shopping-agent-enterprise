# 09. Машиночитаемый формат ответа

## Назначение

Помимо пользовательского Markdown-ответа агент может возвращать
структурированный JSON для интеграций, тестирования и хранения.

Пример расположен в
[`schemas/example-output.json`](../schemas/example-output.json).

## Обязательные поля верхнего уровня

- `schema_version`;
- `query`;
- `checked_at`;
- `result_status`;
- `offers`;
- `alternatives`;
- `unconfirmed_fields`;
- `warnings`.

## Состояние товара

Допустимые значения:

- `new`;
- `used`;
- `refurbished`;
- `unknown`.

## Уровень оригинальности

Допустимые значения:

- `high`;
- `medium`;
- `low`;
- `insufficient_data`.

## Значения риска

Каждый риск содержит:

- код;
- уровень;
- описание;
- подтверждение;
- влияние на итог.

## Денежные значения

Денежные поля содержат:

- значение;
- валюту;
- статус подтверждения;
- источник;
- дату проверки.

Значение `null` используется вместо предположения.

## Результат поиска

`result_status` принимает значения:

- `complete`;
- `partial`;
- `insufficient_data`;
- `no_exact_matches`;
- `error`.

## Совместимость

Изменение обязательных полей требует увеличения основной версии
`schema_version`.
