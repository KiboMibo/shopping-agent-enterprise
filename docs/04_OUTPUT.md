# 04. Формат ответа

## 1. Краткий вывод

В двух–пяти предложениях:

- что удалось идентифицировать;
- какой вариант рекомендуется;
- какие ограничения наиболее важны.

## 2. Параметры поиска

Укажи:

- запрос;
- регион;
- состояние;
- бюджет;
- дату проверки;
- допущения;
- неподтверждённые параметры.

## 3. ТОП-5

Используй таблицу из
[comparison-table.md](../templates/comparison-table.md).

Минимальные столбцы:

- место;
- товар и ссылка;
- состояние;
- продавец;
- цена;
- доставка;
- итог;
- регион отправки;
- оригинальность;
- надёжность;
- ключевой риск.

## 4. Продавцы и отзывы

Для каждого предложения кратко укажи:

- подтверждённый рейтинг;
- число оценок;
- повторяющиеся положительные темы;
- повторяющиеся жалобы;
- ограничения анализа.

## 5. Оригинальность и происхождение

Укажи уровень вероятности и основания. Отдельно обозначь:

- официальный канал;
- параллельный или серый импорт;
- гарантию производителя;
- гарантию продавца;
- возможность проверки серийного номера.

## 6. Риски

Риски делятся на:

- критические;
- существенные;
- умеренные;
- информационные.

## 7. Аналоги

Для каждого аналога укажи:

- отличие;
- совместимость;
- преимущество;
- компромисс;
- ориентир по подтверждённой стоимости.

## 8. Акции

Разделяй:

- гарантированные скидки;
- применимые купоны;
- гарантированный кэшбэк;
- условные предложения;
- неподтверждённые акции.

## 9. Рекомендация

Используй шаблон
[recommendation.md](../templates/recommendation.md).

## 10. Неподтверждённые данные

Заканчивай ответ отдельным перечнем сведений, которые:

- отсутствуют;
- противоречат друг другу;
- требуют проверки при заказе или получении.
