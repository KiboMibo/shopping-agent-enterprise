# 08. Обработка ошибок и недостатка данных

## Общий принцип

Ошибка инструмента, недоступная страница или отсутствие данных не
должны приводить к генерации правдоподобного значения.

## Допустимые статусы

- подтверждено;
- частично подтверждено;
- не подтверждено;
- устарело;
- источник недоступен;
- данные противоречат друг другу;
- неприменимо.

## Недоступная карточка

Если карточка недоступна:

- не использовать цену из памяти;
- не утверждать наличие;
- не включать предложение в основной ТОП-5;
- допускается упомянуть его в разделе неподтверждённых кандидатов.

## Найдено меньше пяти предложений

Агент не должен дополнять ТОП-5 выдуманными или слабо связанными
товарами.

Следует:

1. вывести фактически найденное число точных предложений;
2. объяснить, почему предложений меньше пяти;
3. отдельно показать аналоги;
4. не смешивать аналоги с точными совпадениями.

## Неизвестная доставка

Если доставка не подтверждена:

- итоговая стоимость не считается окончательной;
- поле доставки получает значение «не подтверждено»;
- предложение получает штраф;
- пользователь предупреждается о необходимости проверки корзины.

## Неизвестная цена

Предложение без подтверждённой цены не участвует в сравнении по
стоимости и не должно занимать первое место.

## Ошибка распознавания фотографии

При низкой уверенности агент должен представить несколько гипотез
и запросить проверяемый признак: маркировку, серийный номер, размеры
или дополнительный ракурс.

## Ошибка вычисления

Все денежные компоненты должны иметь одну валюту.

При невозможности конвертации:

- значения не суммируются;
- курс не выдумывается;
- предложения сравниваются отдельно.
