# 02. Поиск и сбор данных

## Стратегия поиска

Поиск выполняется от наиболее точного запроса к расширенному:

1. точный артикул или модель;
2. бренд + модель + модификация;
3. нормализованные варианты написания;
4. ключевые технические характеристики;
5. визуальные признаки;
6. совместимые аналоги.

## Приоритет источников

Для подтверждения происхождения и характеристик:

1. официальный сайт производителя;
2. официальный магазин;
3. авторизованный дилер;
4. крупный маркетплейс;
5. профильный магазин;
6. классифайд;
7. иные источники.

Высокий приоритет источника не отменяет проверку конкретного
продавца и конкретного предложения.

## Обязательные поля кандидата

Для каждого найденного предложения собираются:

| Поле | Требование |
|---|---|
| Название | Полное название варианта |
| URL | Прямая ссылка |
| Цена | Текущая подтверждённая цена |
| Доставка | Стоимость или статус «не подтверждено» |
| Сборы | Обязательные комиссии и платежи |
| Итог | Расчётная итоговая стоимость |
| Наличие | Подтверждено или не подтверждено |
| Регион | Город или регион отправки |
| Состояние | Новый, б/у, восстановленный |
| Продавец | Название или идентификатор |
| Рейтинг продавца | Значение и число оценок |
| Отзывы товара | Рейтинг, объём и основные темы |
| Гарантия | Производитель, продавец или отсутствует |
| Возврат | Сроки и ограничения |
| Комплектация | Подтверждённый состав |
| Оригинальность | Вероятность и основания |
| Дата проверки | Дата актуальности данных |

## Поиск по фотографии

1. Опиши наблюдаемые признаки без преждевременной идентификации.
2. Распознай текст, маркировку и логотипы.
3. Сформируй несколько гипотез.
4. Сопоставь форму, размеры, интерфейсы и детали.
5. Отбрось визуально несовместимые результаты.
6. Укажи уровень уверенности.

## Поиск по артикулу

Проверяй:

- точное совпадение символов;
- дефисы, пробелы и ведущие нули;
- код модификации;
- региональную версию;
- отличие артикула производителя от внутреннего SKU магазина.

## Дедупликация

Дубликатами считаются предложения с одинаковыми:

- продавцом;
- товаром и модификацией;
- состоянием;
- ценой и условиями;
- URL или идентификатором карточки.

Разные продавцы одной карточки маркетплейса не считаются одним
предложением, если условия и риски отличаются.

## Скидки, купоны и кэшбэк

Каждая выгода получает статус:

- гарантированная;
- условная;
- персональная;
- неподтверждённая;
- истёкшая.

В итоговую цену включается только гарантированная и применимая
выгода.

## История цены

История цены используется только при наличии:

- проверяемого источника;
- периода наблюдения;
- даты последнего обновления;
- точного совпадения варианта товара;
- указания, учитывались ли купоны и персональные скидки.
