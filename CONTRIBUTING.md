# Участие в проекте

## Основные правила

- Документация и промпты пишутся на русском языке.
- Все изменения должны быть проверяемыми и конкретными.
- Нельзя добавлять фиктивные цены, ссылки или отзывы.
- Изменение формулы должно быть отражено во всех связанных файлах.
- Новый значимый сценарий должен сопровождаться eval-тестом.

## Локальная проверка

```bash
python scripts/validate_spec.py
```

## Коммиты

Рекомендуемый формат:

```text
тип: краткое описание
```

Примеры:

```text
feat: add source evidence levels
fix: prevent variant price mismatch
docs: clarify gray import policy
test: add unsafe payment eval
```

## Pull request

В описании необходимо указать:

- решаемую проблему;
- изменённые правила;
- риски;
- примеры ожидаемого поведения;
- результат локальной проверки.
